# OpenGPGPU

[![Scala](https://img.shields.io/badge/scala-2.13.12-red.svg)](https://scala-lang.org)
[![Chisel](https://img.shields.io/badge/chisel-6.2.0-blue.svg)](https://www.chisel-lang.org)
[![License: BSD-3-Clause](https://img.shields.io/badge/License-BSD_3--Clause-blue.svg)](https://opensource.org/licenses/BSD-3-Clause)

**OpenGPGPU** 是一个完整的开源 GPGPU 项目，旨在提供详细的、基于 Chisel 语言的 GPGPU 内核时钟精确架构建模。

## ⚠️ 当前开发状态

> **注意：** 目前该项目正处于初始阶段，代码库中主要包含了**Memory 模型（内存模型）**的基础设施，目的是为了建立并验证基础的开发和仿真环境。本项目旨在打造一个完整的 GPGPU 架构，内存模块仅仅是整个项目的第一步。

当前的 Memory 模型为后续的 GPGPU 开发提供了一个包含延迟模拟 (Latency)、突发传输 (Burst Transfer) 以及分页内存管理的内存控制器硬件模型与软件后备模型。它使得系统能够在仿真阶段准确地模拟和追踪真实的物理内存行为。

## 📁 项目目录结构

本项目采用了模块化和分层设计的原则，源代码与测试代码分离，并按照模块进行组织。

```text
OpenGPGPU/
├── src/                    # 源代码目录
│   └── <module_name>/      # 各功能模块源代码（例如 memory）
├── tests/                  # 测试代码总目录
│   ├── unit/               # 单元测试目录
│   │   └── <module_name>/  # 各功能模块的单元测试文件存放处（例如 memory）
│   └── integrate/          # 集成测试目录
│       ├── e2e/            # 全局端到端 (End-to-End) 测试
│       ├── tb_isa/         # ISA 指令流水线子系统集成测试
│       ├── tb_task/        # 任务分发子系统集成测试
│       └── tb_cache/       # 缓存子系统集成测试
├── docs/                   # 项目文档目录
├── utils/                  # 公用工具和辅助类源码
├── build/                  # SBT 构建和测试输出目录
└── project/                # SBT 构建配置文件
```

## ✨ 核心特性

OpenGPGPU 旨在构建一个完整的、时钟精确的开源 GPGPU 架构模型，其核心特性涵盖了从底层硬件逻辑到上层系统架构的多个方面：

*   **时钟精确的微架构模拟**: 基于 Chisel3 硬件构造语言实现，能够提供循环精确 (cycle-accurate) 的行为模拟，确保仿真行为与真实硬件逻辑高度一致。
*   **模块化与可扩展设计**: 系统架构采用高度模块化设计，清晰划分了各个关键子系统（如内存管理、缓存子系统、ISA 指令流水线、任务分发等），便于后续功能的独立开发、测试与持续迭代。
*   **完整的内存与缓存子系统**: 具备从软件后备模型到硬件控制器的完整抽象栈，支持分页内存管理、突发传输 (Burst Transfer)、精确的延迟模拟及越界保护。
*   **指令流水线与计算核心**: 计划实现对典型 GPGPU 指令集的支持，构建灵活的指令派发和多线程计算执行模型。
*   **任务调度与分发架构**: 支持高效的全局和局部工作组调度，实现并发执行机制模拟。
*   **健壮的仿真与测试基础设施**: 内置全面的测试台机制，从底层单元测试到高层端到端集成测试一应俱全，保证各模块稳定可靠。

## 🎯 关键里程碑

*   [x] **Phase 1: 基础设施构建与内存模型**
    *   构建项目基础架构，规范目录与测试标准。
    *   实现支持 64 位地址空间和 4KB 分页管理的底层后备内存分配器。
    *   实现包含状态机、突发传输和延迟模拟的 Chisel 硬件内存控制器模型及完整测试台。
*   [ ] **Phase 2: 缓存子系统与互联总线**
    *   集成 L1/L2 Cache 层级结构与缓存一致性机制。
    *   设计多通道内存控制器和片上互联结构 (Network-on-Chip)。
*   [ ] **Phase 3: 指令执行与核心流水线**
    *   定义和解析目标 GPGPU 指令集架构 (ISA)。
    *   开发取指、解码、调度与执行流水线阶段，支持 SIMT 架构。
*   [ ] **Phase 4: 任务调度与系统集成**
    *   实现顶层任务调度与分发模块。
    *   完成全系统端到端验证，并通过典型 Kernel 程序的仿真测试。

## 🚀 快速开始

本项目使用 `sbt` 作为构建工具。

### 环境要求
- JDK 11 或更高版本
- Scala 2.13.x
- sbt 1.x

### 运行测试

克隆项目后，在根目录执行以下命令即可启动完整的测试流程：

```bash
sbt test
```

执行该命令将编译代码并运行所有的单元测试和集成测试。关于详细的测试用例覆盖情况，请参考 [`tests/TEST_COVERAGE.md`](tests/TEST_COVERAGE.md)。

现有的内存基础架构已经足够用于上层缓存模块或总线协议模块的对接。未来的开发重点包括：
- [ ] 结合 DPI-C 或其他协同仿真工具桥接 `simIO`，以便与 C++ 等外部模拟器交互。

- [ ] 进一步增加乱序请求处理以及多通道 (Multi-channel) 读写控制。
- [ ] 逐步建模 GPGPU 的计算核心流水线及其他架构组件。
